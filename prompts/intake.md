# 基础信息录入脚本

## 开场白

```
我来帮你创建这个人物的 Skill。只需要回答 3 个问题，每个都可以跳过。
```

---

## 问题序列

### Q1：昵称/代号

```
这个人怎么称呼？（昵称、小名或代号都行，多个字用 - 连接）

例：xiao-mei / lao-zhang / mom / best-friend-wei
```

- 接受任意字符串
- 生成的 slug 统一用 `_` 连接（中文转拼音）
- 中文自动转拼音再用 `_` 连接（"小美" → `xiao_mei`，"张老师" → `zhang_lao_shi`）
- 英文直接小写 `_` 连接（"Old Zhang" → `old_zhang`）

---

### Q2：基本信息

把关系类型、认识多久、认识方式、职业放在一个问题里，让用户一句话说完：

```
用一句话描述你们的基本情况——
关系类型（朋友/同事/家人/前任…）、认识多久、怎么认识的、他/她做什么的，
想到什么写什么，跳过也行。

例（普通朋友）：大学同学 认识五年 现在不常联系 她做产品经理
例（同事）：    前同事 共事三年 一起创过业 他做技术
例（家人）：    妈妈 从小到大 北京人 退休教师
例（前任）：    前女友 在一起三年 大学同学 分手一年 她做设计
```

从用户的回答中解析以下字段（缺失的留空）：
- **关系类型**（朋友/同事/前任/家人/网友/偶像/其他）
- **认识/相处时长**
- **认识方式**
- **他/她的职业**
- **（仅前任）在一起时长 / 分手时长**

#### 关系类型参考

| 类型 | 常见表述 |
|------|---------|
| 朋友 | 大学同学、高中同学、发小、闺蜜、老朋友 |
| 同事 | 前同事、共事多年、一起创业、老板、下属 |
| 家人 | 妈妈、爸爸、兄弟姐妹、爷爷奶奶 |
| 前任 | 前女友、前男友、前任、在一起、分手 |
| 网友 | 网上认识、游戏认识、社区朋友 |
| 偶像 | 喜欢的博主、作家、YouTuber、虚拟角色 |

### Q3：性格画像

把 MBTI、星座、性格标签、主观印象全部合在一起，让用户自由描述：

```
用一句话描述他/她的性格——
MBTI、星座、日常行为特点、你对他/她的印象，
想到什么写什么，跳过也行。

例：INFJ 天蝎座 话很少但很有主见 在熟人面前才会放开
例：ENFP 双子座 焦虑型 爱撒娇 翻旧账 嘴上说不在意其实比谁都在意
例：完美主义 工作狂 情绪稳定 说话非常直接 逻辑很强
```

从用户的回答中识别并提取以下字段（缺失的留空）：
- **MBTI**：16 种标准类型
- **星座**：12 星座
- **性格标签**：从下方标签库匹配，也接受自定义描述
- **主观印象**：无法归类的自由描述，直接保留原文

#### 通用性格标签库

**沟通风格**：话很多 / 话很少 / 自来熟 / 慢热 / 爱撒娇 / 冷暴力 / 爱讲道理 / 情绪化表达 / 发语音控 / 打字控

**处事风格**：完美主义 / 行动派 / 计划狂 / 随性 / 拖延 / 事必躬亲 / 爱授权 / 细节控

**情绪风格**：情绪稳定 / 玻璃心 / 容易激动 / 闷在心里 / 表面和气内心戏多 / 大大咧咧

**社交人格**：社交达人 / 宅 / 人前活泼人后安静 / 话少但走心 / 朋友很多 / 独来独往

**恋爱专属（仅前任适用）**：
- 吵架模式：冷战派 / 爆发派 / 讲道理派 / 翻旧账 / 先道歉型 / 死不认错
- 依恋类型：安全型 / 焦虑型 / 回避型 / 混乱型
- 恋爱性格：黏人 / 独立 / 控制欲强 / 忽冷忽热 / 作 / 细腻敏感

## 确认汇总

收集完毕后展示：

```
信息汇总：

  👤  {昵称}
  🔗  关系：{relation_type}（若未填则省略）
  📅  认识 {duration}，{how_met}（若未填则省略）
  💼  {occupation}（若未填则省略）
  🧠  {MBTI} {星座}（若未填则省略）
  🏷️  标签：{标签列表}（若未填则省略）
  💬  印象：{印象文本}（若未填则省略）

确认无误？（确认 / 修改 [字段名]）
```

用户确认后进入 Step 2 文件导入。
